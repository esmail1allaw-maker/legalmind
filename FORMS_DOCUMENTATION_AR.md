interface CaseFormData {

## 🔗 التكامل في الملف الرئيسي

### الاستيرادات الجديدة:
```typescript
import AddClientForm from './components/AddClientForm';
import AddCaseForm from './components/AddCaseForm';
import { Client, CaseRecord } from './types/app';
```

### حالات الـ State الجديدة:
```typescript
const [showAddClientForm, setShowAddClientForm] = useState(false);
const [showAddCaseForm, setShowAddCaseForm] = useState(false);
```

### دوال المعالجة الجديدة:
```typescript
// معالج إضافة عميل جديد
const handleAddClient = (newClient: Client) => {
  setClients(prevClients => [newClient, ...prevClients]);
};

// معالج إضافة قضية جديدة
const handleAddCase = (newCase: CaseRecord) => {
  setCases(prevCases => [newCase, ...prevCases]);
  setClients(prevClients =>
    prevClients.map(client =>
      client.id === newCase.clientId
        ? { ...client, casesCount: client.casesCount + 1 }
        : client
    )
  );
};
```

### ✅ الطريقة الجديدة (مع Supabase):
```
الإدخال 
  ↓

  ↓
إرسال إلى Supabase (مع معالج الأخطاء)
  ↓
استقبال البيانات المحفوظة بـ ID فريد
  ↓
تحديث State المحلي فوراً
  ↓
عرض البيانات الجديدة بدون Refresh
  ↓
رسالة نجاح/خطأ واضحة
```

## ⚡ المزايا الرئيسية

### 1. **التحقق من الصحة (Validation)**
```typescript
// التحقق من أرقام الهاتف اليمنية فقط
const phoneRegex = /^(77|73|71|70)\d{7}$/;
if (!formData.phone.match(phoneRegex)) {
  setValidationError('يرجى إدخال رقم هاتف يمني صحيح');
  return false;
}

// التحقق من صيغة البريد الإلكتروني
const emailRegex = /^[^\s@]+@[^\s@]+\.[^\s@]+$/;
if (!formData.email.match(emailRegex)) {
  setValidationError('صيغة البريد الإلكتروني غير صحيحة');
  return false;
}
```

### 2. **منع الإرسال المتكرر**
```typescript
// جميع الأزرار والحقول معطلة أثناء الإرسال
<button disabled={isSubmitting}>
  {isSubmitting ? (
    <>
      <Loader size={18} className="animate-spin" />
      جاري الحفظ...
    </>
  ) : (
    <>
      <Save size={18} />
      حفظ الموكل
    </>
  )}
</button>
```

### 3. **معالجة الأخطاء الشاملة**
```typescript
try {
  const { data, error } = await supabase
    .from('clients')
    .insert([{ /* البيانات */ }])
    .select()
    .single();
  
  if (error) throw new Error(error.message);
  
  // النجاح
  onClientAdded(data);
  onSuccess('تم إضافة الموكل بنجاح');
  
} catch (error: any) {
  const errorMessage = error.message || 'حدث خطأ';
  onError(errorMessage);
  console.error('خطأ:', error);
}
```

### 4. **التحديث الفوري للـ State**
```typescript
// البيانات الجديدة تظهر مباشرة في القائمة
// بدون الحاجة لـ Refresh الصفحة
setClients(prevClients => [newClient, ...prevClients]);
setCases(prevCases => [newCase, ...prevCases]);
```

## 🚀 طريقة الاستخدام

### إضافة عميل جديد:
1. **اذهب إلى صفحة "العملاء والموكلين"**
2. **اضغط على زر "إضافة عميل / موكل جديد"** (الزر الأصفر في الأعلى)
3. **ملأ النموذج:**
   - اسم الموكل (إلزامي)
   - رقم الهاتف بصيغة يمنية (إلزامي) - مثال: 771234567
   - البريد الإلكتروني (إلزامي) - يجب أن يكون صيغة صحيحة
   - نوع العميل (فرد أو شركة)
4. **اضغط على "حفظ الموكل"**
5. **سيتم حفظ البيانات في Supabase وتحديث القائمة فوراً**

### إضافة قضية جديدة:
1. **اذهب إلى صفحة "القضايا"**
2. **اضغط على زر "فتح ملف قضية جديد"** (الزر الأصفر في الأعلى)
3. **ملأ النموذج:**
   - عنوان القضية (إلزامي)
   - اختر العميل المرتبط (إلزامي) - من القائمة المنسدلة
   - فئة القضية (تجاري، مدني، إلخ)
   - حالة القضية (نشط، تحت الدراسة، إلخ)
   - اسم المحكمة (إلزامي)
   - الرقم القضائي (إلزامي)
   - معرّف المحامي (اختياري)
   - وصف القضية (اختياري)
4. **اضغط على "حفظ القضية"**
5. **سيتم حفظ البيانات وتحديث القوائم فوراً:**
   - إضافة القضية الجديدة إلى قائمة القضايا
   - زيادة عدد القضايا النشطة للعميل
